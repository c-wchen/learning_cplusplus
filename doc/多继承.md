## 多继承(实际开发中不建议使用)
```c
class A {}
class B {}
class C: public A, public B {}
```
1. 二义性
    > 多个父类出现的同名的成员，需要添加作用域
2. 菱形（钻石）继承
    > B、C都会保存A的实例，当D访问继承下来的A（虚基类）数据时会出错。
    子类继承两份相同的数据，导致资源浪费和毫无意义
```bash
                    A
                   / \
                  B   C     
                   \ /
                    D
```
